# bm-blocked

Локальный Windows-сервис для проверки заблокированных площадок активных кампаний РСЯ в Яндекс Директе.

## Установка

1. Откройте раздел **Releases** на странице репозитория.
2. Скачайте `bm-blocked.zip` из последнего выпуска.
3. Полностью распакуйте архив в отдельную папку.
4. Запустите `bm-blocked.exe`.
5. Введите общий логин и пароль компании.

Приложение откроет интерфейс в браузере и продолжит работать в системном трее. Для завершения используйте пункт **Выход** в меню значка трея.

Не переносите EXE отдельно от остальных файлов: рядом должны оставаться `server.js`, `index.html`, `auth-config.json` и папка `runtime`.

## Возможности

- загрузка клиентов Яндекс Директа по OAuth-токену;
- получение активных кампаний с показами в РСЯ;
- проверка доступности web-площадок;
- пропуск app-площадок без попыток открыть их как сайты;
- удаление подтверждённо нерабочих площадок из выбранных кампаний;
- системное уведомление после долгой проверки, когда страница неактивна;
- светлая и тёмная темы.

OAuth-токен не записывается в файл и используется только локальным сервером для запросов к API Яндекс Директа.

## Запуск из исходников

Потребуется Node.js 18 или новее:

```powershell
node server.js
```

После запуска откройте [http://127.0.0.1:8124/index.html](http://127.0.0.1:8124/index.html).

## Сборка Windows-версии

```powershell
powershell -ExecutionPolicy Bypass -File .\build-exe.ps1
```

Готовый комплект появится в `dist`, архив для публикации — в `bm-blocked.zip`.

## Авторизация

Учётные данные не хранятся в исходниках. `auth-config.json` содержит отдельные PBKDF2-SHA256-хеши логина и пароля. Файл исключён из Git и добавляется только в готовую сборку.

Для создания новой конфигурации:

```powershell
powershell -ExecutionPolicy Bypass -File .\set-auth.ps1
```
